Every time I look at the AIS800-64O, it’s never fully populated… A full set of transceivers costs more than the switch itself.

Lately I’ve often met people who don’t know how to install SONiC. To save time sending the same instructions over and over, here’s a simple walkthrough.

First, you need to know about ONIE. I previously already wrote an ONIE introduction.

There are two main ways to install or upgrade SONiC:

  • USB
  • HTTP
  • sonic-installer (typically used for upgrades)

Let’s get started.

USB

Personally, this is the easiest and laziest method — basically “just leave it alone” Steps:

1. Format the USB drive to FAT

On macOS, in Disk Utility choose View → Show All Devices to see the whole USB device (not just the partition).

Choose MS-DOS (FAT) and MBR. For Windows, please do it your preferred way.

2. Put the image on the USB and name it onie-installer.bin

It’s already 2026, but safely ejecting your USB is still a good idea.

3. Plug the USB into the switch and power it on. Two scenarios:

No NOS installed / brand-new device

This is the simplest. Truly “just leave it alone” — it will auto-install. Just wait.

A NOS is already installed and not removed

If there’s already a NOS, you may need to remove it first. If installation isn’t allowed, ONIE will remind you to uninstall first.

3-1. In the ONIE menu, select “Install OS”
3-2. ONIE will auto-install

If the device only has a Type‑C USB port, you can use a USB hub. In the screenshot below, the highlighted area shows ONIE detecting the USB, auto-mounting it, and installing automatically.

If USB detection fails and you want to retry, run onie-stop and onie-start to restart auto-discovery.

3-3. Wait for installation to finish; the device will reboot to the login screen

HTTP

This method requires an HTTP server. If conditions allow, you can also set up DHCP and a ZTP server.

1. In the ONIE menu, select “Install OS”

2. ZTP Auto Discovery Mode

If you have a ZTP environment ready, again just “just leave it alone” — it will auto-install. Otherwise, proceed to the next step.

3. Manually enter the image URL

First, stop ZTP so it doesn’t spam your console.

If there’s no DHCP, set an IP address manually and verify you can reach the HTTP server.

Currently ONIE cannot allow files via SCP from outside.

Run the install command manually: onie-nos-install http://{IP}/{URL}/{Filename}

You can also curl the file first and then run the same install command with a local path, e.g., onie-nos-install /tmp/{Filename}

Wait for installation to finish; the device will reboot automatically. That’s it — just wait for the system to boot.

After installation, you’ll see the login screen:

2. Install command

Run the install as shown. After that, use sudo sonic-installer list to verify the second NOS entry appears.

Changing / Switching NOS

First, run sudo sonic-installer list to see available NOS entries. If you don’t want to switch via GRUB manually on the console, you can use sonic-installer to switch.

You can also use set-default to choose the default NOS for future boots.

Upgrade SONiC version

As you can see from the above, it’s basically “install then switch.” Nothing much more to say.

Remove a NOS

Removing a NOS with sonic-installer deletes the other NOS. For example, if you have NOS-A and NOS-B on disk, and you’re currently running NOS-A, then running sonic-installer remove or sonic-installer cleanup will remove the other one (NOS-B).

Cheatsheet — Quick HTTP server on macOS

When helping a customer onsite without an existing HTTP server, this trick works great. Just place files under /Library/WebServer/Documents (use sudo to copy files there). Of course, your Mac and the switch must be connected and have IPs set. A simple RJ45 direct connection between Mac and switch with static IPs will do.
