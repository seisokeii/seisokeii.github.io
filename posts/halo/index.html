Install Hugo

Here is a very rough explanation of how I got Hugo up and running~

  1. Install Hugo, Go (I won’t go into details here, please check it yourself)
  2. Create a new website , I chose submodule here because it is easier to update the theme later. Hugo seems to have mod that can be used, you need to read the documentation for this 
    hugo new site my_website
cd my_website
git init
git clone https://github.com/dillonzq/LoveIt.git themes/LoveIt
git submodule add https://github.com/dillonzq/LoveIt.git themes/LoveIt
  3. Add theme = 'LoveIt' in hugo.toml, and this LoveIt is the directory name placed in the last command of step 2 You can also directly use the config.toml from exampleSite. Whether you want to rename it to hugo.toml is up to you
  4. View the result, execute the command under the my_website directory (the so-called root directory) 
    hugo server
  5. Next, you can start writing articles and adjusting the settings in hugo.toml. I won’t go into details here because I' m still exploring…

Github Action

There is a very cool point to mention here, that is, the blog can put the entire Hugo project in a private repository, and then deploy it to the github.io project through Github Action’s workflows. This way, you can hide the source code and some bad things^^ (Thanks to CSY for teaching me this trick)

Here’s how to do it:

  1. Create a .github/workflows/deploy.yml file in the Hugo project
  2. Take my deploy.yml file as an example. Please think about the things I need to replace in it yourself. It shouldn’t be difficult, right!
---
name: Deploy public directory to seisokeii.github.io (MyBlog) repository

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3
        with:
          token: ${{ secrets.<GITHUB PAT> }}

      - name: Setup SSH
        uses: MrSquaare/ssh-setup-action@v1
        with:
          host: github.com
          private-key: ${{ secrets.<Your SSH private key> }}

      - name: Copy public directory
        run: |
          mkdir -p target-repo
          git clone git@github.com:seisokeii/seisokeii.github.io.git target-repo
          cp -r public/* target-repo/

      - name: Commit and push changes
        run: |
          cd target-repo
          git config user.name Userabc
          git config user.email User@mail
          git add .
          SukaBlyad_date=$(date +%Y/%m/%d) # This is just for commit message
          git commit -m "Update: $SukaBlyad_date"
          git push
  1. After having the above YAML configuration file, remember to set up the necessary things, such as PAT (Personal Access Token) and SSH private key. I generated these things and put them in the environment variables of the project (repo Settings -> Secrets and Variables -> Actions). Remember that it only allows you to update, not to view existing ones, so remember to save them first!
  2. After completing the above, try to push the entire Hugo project and see if GitHub Action can succeed and if only the public things appear in the github.io project.
