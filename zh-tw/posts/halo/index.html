第一個貼文就送給 hugo 吧！

首先，不得不說偶的運氣真的差的可憐… 看了網路很多文章都寫說 hugo 5分鐘就可以架設好了，結果窩花了一天的時間(20小時以上)才搞定開始寫這則貼文…

至於原因嘛… 一開始就挑酷酷的主題去使用，結果呢幾乎每個主題都只說到 git submodule 安裝這邊就結束了，剩下的就是請自己想辦法，對於懂的人來說這倒是真的，但是對於一開始的超級小白就不好說了，尤其是不會去看 Document 的人，這邊就是我盒盒盒。

窩遇到的問題其實很簡單，看著錯誤訊息想了老半天才發覺，就是在設定檔內少了或打錯 theme = 'xxx' 這東東 = =|||

把這個設定加上去後的確 5 分鐘可以搞定，但是還不包含整體環境設定的修改，因為主題的作者都叫你去 exampleSite 直接下指令跑起來。 這就是第二個問題，小白不會一開始就知道 config.toml (現在改成 hugo.toml) 是怎麼搞的，要吃哪裡的設定檔啥的…

大致這兩點問題就是不看 Document 的人，照著說明直接坐下去有機會遇到的吧，事後還是要慢慢的去看一下 Document 唄！ 但是窩就是懶，而且窩其實超級不喜歡網頁相關的東西，但是窩很崇拜寫網頁的人，因為網頁相關的工具技術太多且更新太快，窩根本跟不上…

B.T.W. GitHub Page (GH) 也就是 github.io 這個專案中，只能放 public 裡面的東西，因為 GH 會去讀取該專案下的 indel.html 檔。 如果是將整個 hugo push 上去的話會跑不動，就算你加了 .nojekyll 讓 deploy 成功也一樣會得到 404 的訊息盒盒盒 (窩不會說這是偶遇到過的)

安裝 Hugo

這邊就超級粗略的講一下這邊 hugo 是怎麼被窩搞起來的吧～

  1. 安裝 hugo、Go (這個就不多說了，請自己查唄)
  2. 創建新的網站 ，這邊我選擇 submodule 是因為之後主題更新比較簡單，hugo 好像有 mod 可以用，這要去看文件教學了

  3. 在 hugo.toml 內加入 theme = 'LoveIt'，而這個 LoveIt 就是第2點最後一個指令所放的目錄名稱 其實也可以直接拿 exampleSite 的 config.toml 來用，要不要把檔名改成 hugo.toml 就看你了
  4. 查看結果，指令請在 my_website 目錄下執行（所謂的根目錄） 
    1

    hugo server
  5. 接下來就可以開始寫文章了跟調整 hugo.toml 的設定檔了，這邊就不多說了，因為窩也還在摸索中…

Github Action

這裡有個很酷的點要提一下，就是部落格可以將整個 Hugo 專案放在一個 Private 專案中，然後透過 Github Action 的 workflows 部署到 github.io 專案中，醬可以把原始碼跟一些壞壞東西給藏起來＾＾ （感謝 CSY 教我這一招）

做法的話呢如下：

  1. 在 hugo 專案中新增 .github/workflows/deploy.yml 檔案
  2. 以偶的 deploy.yml 檔案為例，我裡面要替換的東西就請自行思考了，應該不難吧！

---
name: Deploy public directory to seisokeii.github.io (MyBlog) repository

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3
        with:
          token:  ${{ secrets.你的吉哈伯PAT }}

      - name: Setup SSH
        uses: MrSquaare/ssh-setup-action@v1
        with:
            host: github.com
            private-key: ${{ secrets.你的SSH private key }}

      - name: Copy public directory
        run: |
          mkdir -p target-repo
          git clone git@github.com:seisokeii/seisokeii.github.io.git target-repo
          cp -r public/* target-repo/          

      - name: Commit and push changes
        run: |
          cd target-repo
          git config user.name 使用者abc
          git config user.email 信箱abc
          git add .
          SukaBlyad_date=$(date +%Y/%m/%d) # 這邊我就只是記錄啥時候更改而已，可以不用學我使用日期，可以看我的 commit comment
          git commit -m "Update: $SukaBlyad_date"
          git push
  1. 有了以上 YAML 設定檔後，要記得去設定所需要的東西，像是 PAT (Personal Access Token) 跟 SSH private key，我是將這些東西產生好後放在專案中的環境變數(repo Settings -> Secrets and Variables -> Actions)，記得它只給你 Update 而已，不會顯示現有的，所以要記得先儲存好呦！
  2. 完成以上後，嘗試 push 整個 hugo 專案，看看 GitHub Action 能不能成功，以及 github.io 專案是否只有出現 public 的東西。
