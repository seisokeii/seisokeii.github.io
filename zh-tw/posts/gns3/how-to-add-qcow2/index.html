感覺網工的控制欲是不是都很強啊？ 不管現在已經有很多現成的東西了，最後還是想要客製化成自己想要的形狀… 還是說每個人都這樣？？？

因為要來做一下 Datacenter AI Network 的拓樸，然後每次都覺得 template 要改東改西的好麻煩。 主要是因為現在 SXM 架構或 OAM server (GPU 焊在 PCB 上) 都是 8 張 NIC 是直通 GPU，然後還會有 4 張 NIC 給 internet 跟 storage (左右 CPU)，還有像是 RJ45 這種 management or BMC 用的。 有一拖拉庫的網卡… 因為比較常碰到這類的 case 所以就想說來製作一個模板放在 GNS3 上面模擬一些基本的 routing protocol 拓樸驗證。

1. 建立 VM

這邊教的製作 qcow2 主要是要從 vmdk 去轉換，所以先安裝一台 VM 並且在該 VM 中先做好需要的設定以及預先安裝需要的套件。

記得 VM 要選擇儲存檔案的方式要是 single file，不要分散很多檔案嘿

VM 整個都設定好後就使用 sudo poweroff 指令完整關機。 這邊應該就不用多說明了吧～

這邊還是舉個例子吧XD

範例需求

這個是最後 qcow2 要在 GNS3 上面的規格，但是 VM 先不需要建立 13 張網卡，只需要建立一張即可，到時候再 GNS3 建立模板時候再加就好。

  • 13 張網卡 (gpu-0~8, mgmt, net, storage)
  • 2 cores
  • 4096 MB

範例 VM 安裝相關套件

這邊就是看要之後需要什麼就可以先安裝到 VM。

sudo apt update

範例 VM 預設定 - NIC rename script

因為我想要 qcow2 開進去後網卡都被命名，所以先放了一個腳本在 VM 裡面，然後梗改一下 motd 未來開機後就可以有提示指令，就可以不用在那邊翻文件了。

sudo vi /usr/local/sbin/create-nic-links.sh

然後貼上以下內容

#!/bin/bash
set -e

names=(
  mgmt-1
  net-1
  net-2
  storage-1
  storage-2
  gpu-1
  gpu-2
  gpu-3
  gpu-4
  gpu-5
  gpu-6
  gpu-7
  gpu-8
)

mkdir -p /etc/systemd/network

mapfile -t ifaces < <(
  find /sys/class/net -mindepth 1 -maxdepth 1 \
  ! -name lo -printf '%f\n' | sort -V
)

echo "找到 ${#ifaces[@]} 張網卡"

if [ "${#ifaces[@]}" -ne 13 ]; then
  echo "錯誤：目前不是 13 張網卡"
  printf '%s\n' "${ifaces[@]}"
  exit 1
fi

rm -f /etc/systemd/network/10-ai-*.link

for i in "${!names[@]}"; do
  iface="${ifaces[$i]}"
  path=$(udevadm info --query=property \
    --path="/sys/class/net/$iface" |
    sed -n 's/^ID_PATH=//p')

  echo "$iface -> ${names[$i]} ($path)"

  cat > "/etc/systemd/network/10-ai-${i}.link" <<EOF
[Match]
Path=$path

[Link]
Name=${names[$i]}
EOF
done

update-initramfs -u
reboot

MOTD 腳本

sudo vi /etc/update-motd.d/99-ai-server

貼上以下到 99-ai-server motd 檔案

#!/bin/bash

if [ ! -f /etc/systemd/network/10-ai-0.link ]; then
    echo
    echo "==================================================="
    echo " First time boot need to execute following command"
    echo
    echo " sudo /usr/local/sbin/create-nic-links.sh"
    echo "==================================================="
    echo
fi

新增執行權限給 motd 檔案

sudo chmod +x /etc/update-motd.d/99-ai-server

試跑一下

run-parts /etc/update-motd.d/

sudo chmod 600 /etc/netplan/99-ai-nics.yaml
sudo netplan generate

差不多這樣就可以關機準備轉換成 qcow2 了。

2. 從 VMDK 轉換成 qcow2

去到 VM 檔案的目錄，應該可以看到 vmdk 這一個檔案，在這個目錄底下做轉換唄。

# 從 vmdk 轉換成 qcow2
qemu-img convert -p -f vmdk -O qcow2 AI-server-emulate.vmdk ai-server.qcow2

查看 qcow2 的資訊

qemu-img info ai-server-small.qcow2

壓縮 qcow2 檔案大小，因為我製作出來的 qcow2 大約 5GB… 每次都傳好久所以想要壓縮一下

qemu-img convert -p -c -f qcow2 -O qcow2 ai-server.qcow2 ai-server-small.qcow2

3. 上傳 qcow2 到 GNS3 server

目前我的 GNS3 server 上面放 QEMU 的檔案是在 /home/gns3/GNS3/images/QEMU/ 底下

scp ai-server.qcow2 {username}@{GNS3_IP}:/home/gns3/GNS3/images/QEMU/

4. 新增 GNS3 QEMU template

找到 Preferences 設定頁面 Edit(GNS3) -> Preferences -> QEMU VMs -> New

依序設定：

  • QEMU 名稱
  • 記憶體 (4096 MB)
  • Console type (選擇 VNC)
  • 選擇 Disk image，這邊選擇剛剛上傳的 ai-server.qcow2
  • Finish

然後編輯一下該 QEMU 去做對應的修改，以及養成好習慣寫好說明方便後續其他人跟未來失憶的自己使用。

其實到目前這樣就差不多了，之後要自己製作想要的 qemu 可以仿照一下這些步驟去製作～ 不要老是當伸手族
