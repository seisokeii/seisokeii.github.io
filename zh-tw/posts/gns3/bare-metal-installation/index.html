用過了好多模擬器 (GNS3, EVE-ng, PNETLab, CML, packet bug-it tracer …) 感覺還是變色龍好用😈 俗話說的好免錢的最貴… 但是我倒是覺得其他模擬器付費後好像服務也不是很優…

想必每個學網路的人應該幾乎都會使用模擬器… 應該吧XD 沒用過的應該都是有錢人直接買機器來玩齁。 想當初我還特地請朋友幫我拖一台 Dell R6515 回來為了之後在家做 lab 用的，因為當時要畢業離校了已經沒有 vCenter 可以蹭了QQ

結果我到公司上班後突然就把整個 lab 都讓我管，所以我就在那邊東拼西湊一些公司曾經做的 EOL/EOS 的 server 來當作新的模擬機～ 可惜還是自己買了更多 core 的 CPU 跟一堆記憶體… 花了好多錢錢又不給報帳QQ

這邊就做記錄一下 GNS3 bare metal 要怎麼安裝吧～ 可以先參考 GNS3 官方安裝文件

記得很久以前都是官方給一個 VMDK 檔案還啥的要我們裝成虛擬機，後來官網好像才出現 bare metal 的方法，也可能是我小時候沒好好看文件XD

Server Hardware 規格

就拿我現在機器的 Spec 來當例子吧:

  • Ubuntu LTS version. (24.04.2)
  • CPU: 6 core * 2 sockets (E5-2603 v4 @ 1.70GHz)
  • Memory: 160 GB
  • HDD: 1024 GB

1. Install VM

第一步就先記得去裝好一台 Ubuntu server 吧 這邊應該就不多寫教學了…

有了 VM 後先進去建立一個 GNS3 要使用的帳號

# 新增使用者與家目錄
sudo useradd gns3 --create-home

# 改密碼
sudo passwd gns3

# 添加一下相關權限
sudo usermod -aG ubridge,libvirt,kvm,docker gns3

2. 製作 Daemon

之前是照其他文章博主去將 gns3server fork 到背景執行，然後 systemd 要靠 PID file 找主程序，後來遇到 PID file 不見還怎樣的就會死去。 所以就改成目前這樣子了。

去 /etc/systemd/system/ 建立一個 gns3.service 檔案。

sudo vi /etc/systemd/system/gns3.service

貼上以下內容

[Unit]
Description=GNS3 server
After=network-online.target

[Service]
Type=simple
User=gns3
Group=gns3
ExecStart=/usr/bin/gns3server --log /var/log/gns3/gns3.log
Restart=on-failure
RestartSec=5
LimitNOFILE=16384

[Install]
WantedBy=multi-user.target

建立完後記得改一下檔案權限

sudo chown root:root /etc/systemd/system/gns3.service
sudo chmod 644 /etc/systemd/system/gns3.service

3. 讓 GNS3 服務跑起來

剛剛已經將 systemd 需要的檔案建立好了，接下來就是要讓系統認到跟啟動。

# 讓 systemd 重新讀一次
sudo systemctl daemon-reload

# enable + start 一起跑
sudo systemctl enable --now gns3

檢查服務

# 查看服務本身狀態資訊
sudo systemctl status gns3 --no-pager

# 查看 socket port 是否有開放 0.0.0.0:3080
sudo ss -lntp | grep 3080

如果有錯誤請查看 log

sudo journalctl -u gns3 -n 100 --no-pager

升級 bare metal 的 GNS3

可以用這個指令確認現在套件的來源與可以升級的資訊

apt-cache policy gns3-server gns3-gui gns3-iou

升級指令如下

sudo systemctl stop gns3

sudo apt update

apt list --upgradable | grep -i gns3

sudo apt install --only-upgrade gns3-server gns3-gui gns3-iou

sudo systemctl daemon-reload
sudo systemctl start gns3

sudo systemctl status gns3 --no-pager
gns3server --version

啊如果怕遠端升級的話，可以直接去 console 登入 gns3 帳號，然後就會有 dialog 的介面，那邊可以按升級比較無腦一點，但是就是要 console。

其實大致就這樣而已沒有到很難，多裝個幾次就熟了XDDD

關於 images 的部分就要請大家各憑本事去拿了 像是花個幾塊錢去百度網盤買 VIP 之類的，有大大提供超級多 image 那個查一下就知道了～

這邊我就沒有去寫 IOU license 啥之類的教學了，因為現在 cisco 他們有允許帳號是可以去免費下載他們的 vIOS, vNXOS, … 之類的特定 image， juniper 其實也有提供 vMX, vEX 可以下載，這些都非常貼近真實機器的指令與功能，然後也可以去 community SONiC 去下載 sonic-vs.img 雖然沒很好用但是足矣。

至於有些 images 有些功能沒辦法運作或是啥的，其實大多是因為 switch 對於封包處理都是給 ASIC 晶片，而模擬器上沒有呀，所以其實是 NOS 他們用 CPU 去模擬該功能， 舉例像是 SONiC 的 ACL 功能，這個就是依賴 MAC(ASIC) 去做，然後 community 沒有人去把這個功能使用 CPU 去模擬打包成 sonic-vs.img，這塊 cisco/juniper 應該沒這問題。

大致就是醬子～ 以後有機會再來多分享一些拓樸跟協定的實作吧～
