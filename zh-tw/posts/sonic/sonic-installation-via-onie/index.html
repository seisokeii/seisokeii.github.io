每次看到 AIS800-64O 都不會是接滿的狀態… Transceiver 接滿的時候比一台機器還貴…

近期很常遇到說不會安裝 SONiC 的人，想說就寫了一篇教學以後省得每次在傳教學文件…

首先要知道 ONIE 這個東東，在先前有寫過一篇 ONIE 介紹 了， 小弟對於這個東東又愛又恨QQ 就不在這裡多說什麼了

關於 SONiC 安裝背後主要就是前兩種方法:

  • USB
  • HTTP
  • sonic-installer (通常是做升級用)

接下來就講一下怎麼做吧～

USB

個人覺得用這個最簡單最懶惰，純純的放置 play 就好。步驟如下：

1. USB 格式化成 FAT

Macbook 的 Disk Utility 可以選擇左上角的 View 然後 Show All Devices 就可以看到整個 USB 而不是 Partition

請選擇 MS-DOS (FAT) 與 MBR
Windows 的話自己想辦法不想寫^^

2. 把 Image 放到 USB 然後命名成 onie-installer.bin

雖然已經 2026 了，但還是建議安全退出 USB 吧～

3. USB 插到 Switch 然後開機，有兩種情況:

沒 NOS 或是全新機器

這個最簡單，就是真的放置 play 就會自動安裝好了，就等待即可~~~

已裝有 NOS 並且未解除安裝

如果原本就裝有 NOS，就看要不要先解除安裝或是直接安裝，反正不能安裝的話會提醒要先解除安裝～

3-1. ONIE 選單選擇 Install OS
3-2. 進入 ONIE 後自動安裝

如果機器只有 Type-C USB 其實可以多用一個 USB HUB 也可以
下圖中框框的部分就是偵測到 USB 然後自動掛載並且安裝，完全自動化。

如果無法偵測到 USB 想要重新偵測，可以手動執行 onie-stop 與 onie-start 來重啟自動偵測

3-3. 等待安裝後重開機進入登入畫面

HTTP

此方法需要有 FTP Server，條件好的可以再多架 DHCP 與 ZTP Server。

1. ONIE 選單選擇 Install OS

2. ZTP Auto Discovery Mode

在這個階段如果有架設好 ZTP 環境的話也一樣放置 play 等待自動安裝即可，如果沒有的話就看下一步

3. 手動輸入 FTP Image URL

首先先停止 ZTP 干擾你的 console 畫面

如果沒有 DHCP 的話，手動輸入 IP，可以戳一下看能不能碰到 FTP Server

目前 ONIE 無法從外部使用 SCP 將檔案放到 ONIE 環境中

手動敲安裝指令 onie-nos-install http://{IP}/{URL}/{Filename}

也可以使用 curl 指令抓取，然後安裝指令一樣，只是改路徑就好，像是 onie-nos-install /tmp/{Filename}

等待安裝完後會自動重開機，到此就結束了，只要等待開機進系統即可～

安裝完成後登入畫面

如果沒有自動安裝，可以自行掛載然後將 URL 改成 image 的絕對路徑即可。
mount -t vfat /dev/sd? /mnt/usb，可以使用 blkid 查看 /dev/sd? 是哪個

刪除 NOS

使用 sonic-installer 刪除 NOS 的方式是刪除另一個 NOS，假如硬碟有 NOS-A, NOS-B 也就是你現在是跑 NOS-A 然後你下 sonic-installer remove/cleanup 的話就是刪掉另一個 NOS-B

Cheatsheet - Macbook HTTP & DHCP Server 架設

當外出幫客人裝系統的時候如果沒有 HTTP Server 就可以用這招~
